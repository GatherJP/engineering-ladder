## Commending Bold Challenges
**プロダクトや組織全体に影響を与える革新的なソリューションを生み出している。特定の領域において、前例がないようなアイデアで新しいプロダクトや機能の変革を起こせる。**

**リスクを恐れて行動しない状況から抜け出し、まず実行して次の判断に繋げることを会社全体に後押ししている。失敗から学ぶ姿勢を促進している。**

**会社全体に広く影響し、前例がなく難易度の高い状況でも、行動することの重要性を発信しチャレンジをリードしている。エンジニアがコンフォートゾーンを抜け出すための環境作りに貢献している。**

### [Engineering Manager Skills]

会社全体の課題を発見し、解決を推進する方法を考えて実施している。

組織課題のためのチャレンジングな会社目標設定に影響力を持っており、組織全体の状況を考慮しながら会社のパフォーマンスを加速できる。

会社全体で失敗を恐れずチャレンジする文化の醸成に貢献している。


## Vision
**会社の目標を達成するため、会社全体で取り組むべきアクションを考え、実行に向けて各チームやPJに働きかけている。**

**会社の組織戦略をエンジニアが理解し合意できるようにリードするなど、目標へ向かって会社内のエンジニアが協力できるように取り組んでいる。**

**会社規模の開発で開発効率の向上や改善のため調査や提案を行い、会社全体で改善やベストプラクティスとしてまとめるための取り組みをリードしている。**

### [Engineering Manager Skills]

会社の目標とロードマップを設定し、他のEMやリーダーが戦略を理解し目標設定を行うことをサポートしている。

エンジニアリングにおけるリーダー達と協力し、将来的な大きな戦略を考えている。

個人ではなくプロセスに対して課題分析することを会社全体へ促し、心理的安全性の保たれた文化の醸成に貢献している。


## Focus on the Mission
**会社規模の開発で、プロダクト全体に関わる技術方針や仕様の決定に関わり、プロダクトの価値を向上させている。長期的な価値向上を見据え、ビジネスゴールの達成をリードしている。**

**プロダクトとビジネス戦略を深く理解し、市場へ与えるインパクトとビジネスゴール達成を見据え、ドメイン同士の関わり方や役割の検討に関わっている。ドメイン知識のエキスパートとして周囲から認められている。**

**長期的に生産性を高めるための深い知識と経験を持ち、それらを踏まえた開発や発信をすることで、会社全体の生産性向上に貢献している。**

### [Engineering Manager Skills]

会社規模での業務やビジネス上の意思決定、それらがお客さまへの価値提供にどのような影響を与えうるかを把握し、また周囲の理解を促進している。

会社全体で他のEMに対してクオリティマネジメントのサポートをしている。品質の維持・向上に努めながら、企業価値を常に提供するために会社横断的な行動を起こしている。

会社規模の開発で、成果とコストのバランスを考慮したうえで大きな成果をだしている。


## Teamwork
**会社内のエンジニアに対し、建設的なフィードバックや称賛によって成長を促進しており、フィードバックの仕方においてロールモデルといえるような人物である。エンジニアがお互いの強みを活かし高め合う文化を醸成している。**

**会社規模の開発で説明責任を果たし、周囲の理解と協力を得る中で、信頼関係を構築している。社内外問わず幅広い信頼関係を構築し、採用などを通して強いエンジニア組織の構築に貢献している。**

**会社規模の開発で、データや信頼できる事実に基づき、事業に良い影響を与えるための意思決定をしている。**

### [Engineering Manager Skills]

オープンで相手を尊重する姿勢でメンバーや他のEMを指導しており、会社規模でエンジニアのオンボーディング改善を行なっている。新しいメンターの育成に取り組み、他のEMに対してキャリア成長の環境やスキルを学ぶ機会を提供できる。

課題解決のために、会社全体のさまざまなメンバーやプロジェクトを繋げられる。

お客さまへ最大の価値を提供するような会社規模の技術的な意思決定を導くため、最も良い指標（時間・コスト・工数）を設定し活用している。


## Professionalism
**会社内の文書化やプロセスの標準化、情報・知識共有の促進に貢献し、それらを文化として醸成している。**
**例：会社全体へベストプラクティスを提案して実行したり、これらを実行および改善するプロジェクトや委員会を主導して貢献する**

リスク調査を行い、会社内で連携して必要な対応ができている。会社内のリスクマネジメントの意識を高め、リスクマネジメントの専門家として認知されている。

**会社内の開発チーム間のコンフリクトを解決するなど、会社規模の開発でステークホルダーが抱える懸念を考慮した解決策に導いている。部門間の対立が起きないような仕組みを構築している。**

### [Engineering Manager Skills]

情報共有プロセスの標準化に貢献し、情報・知識の共有を促進することで、会社内で知識が効果的に活用される環境を整えている。

会社のエンジニアがそれぞれの業務に対してオーナーシップを持つよう促している。

会社内で、自分と異なる意見や多様な考え方を尊重することを促し、メンバーの意見を取り入れて解決策を考えられる。


## Continued Learning
**高度な専門技術や知識を持ち発揮している。専門家レベルの技術や最先端の研究など、プロダクトの価値向上と組織全体の発展のため新しいことを学び実践している。**

**会社全体のエンジニアに対して、自分の役割の範囲に限らず新しい知識を学び実践することや技術情報の発信を促進し、その模範となる行動をしている。**

**他のエンジニアの模範となるようなコードを書ける。特定の分野のコーディングにおいて、エキスパートとして周囲から認められている。**
**コードレビューや体系的なデバッグに関する基準を定めたり、コーディングにおける問題解決や難易度の高いプロジェクトのために助言している。**

**アーキテクトとして幅広く認知されており、ソフトウェアデザインのエキスパートである。開発言語やフレームワーク、ライブラリー、デザインパターン、社内の基盤ドメインなどのエキスパートである。期待されているユースケースや運用を見据えた設計ができ、会社規模の開発で設計をリードしている。**

### [Engineering Manager Skills]

ステークホルダーや期限を戦略的に管理し、エンジニアの自己研鑽のために、会社全体で時間を確保するなど学習の文化を醸成している。

戦略策定、長期のゴール設定、長期のビジョン設定、マネージャー達のマネジメント、効果的なコーチングといったものに関する、経営レベルのスキルを学び実践している。

社内外でエンジニアの成長を促すための環境を作るために、強いリーダーシップを見せている。


## Move Fast for Engineers
**会社全体の技術戦略の策定に寄与し、プロダクト開発のスピードと品質を最大化している。**

**開発効率を改善するために、会社全体の共通プラットフォームの戦略を確立するなど、持続的な生産性向上に貢献している。**

**会社全体において、データや信頼できる事実に基づき、事業に良い影響を与えるための迅速な技術的意思決定に寄与している。**

### [Engineering Manager Skills]

会社全体で効率的な開発を実現するための施策を推進している。

会社全体で持続的に生産性を向上するための組織づくりに貢献している。

会社全体に関わる戦略的な意思決定の仕組みを確立している。お客さまへ最大の価値を提供するため、会社規模の技術的な意思決定を導く最適な指標（時間・コスト・工数）を設定し活用している。
